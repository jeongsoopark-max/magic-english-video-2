# Magic English — 통합 홈페이지 + 화상 수업

기존 Google Sites(magic-english-korea)에 있던 홈페이지 내용을 그대로 옮기고, 그 위에 자체 화상 수업 시스템을 연결한 통합 사이트입니다. 배포 주소 하나로 홈페이지와 화상 수업이 모두 동작합니다.

구성:
- **server.js** — Node.js + Express + Socket.io. 화상 수업 참가자끼리 WebRTC 연결을 맺도록 "중개"만 하고, 실제 영상/음성은 서버를 거치지 않고 참가자끼리 직접 주고받습니다(P2P).
- **public/index.html** — 홈페이지("/" 로 접속하면 이 화면). 기존 Google Sites 임베드에 있던 내용(5대 맞춤 클래스, 자료실 허브, 1:1 질문하기, 선생님 모드)을 그대로 옮겼습니다.
- **public/class.html** — 화상 수업 입장 화면(이름/수업 코드 입력 후 입장). 홈페이지의 각 클래스 카드에서 "화상 수업 입장" 버튼을 누르면 여기로 연결됩니다.
- **public/client.js / style.css** — 화상 수업 화면의 동작(WebRTC, 채팅, 화면 공유)과 스타일.

## 1. 로컬에서 실행해보기

사전 준비: [Node.js](https://nodejs.org) 18 버전 이상.

```bash
cd video-class
npm install
npm start
```

브라우저에서 `http://localhost:3000` 접속 → 홈페이지가 뜹니다. "5대 맞춤 클래스"에서 클래스 카드를 클릭 → 모달에서 "화상 수업 입장"을 누르면 수업 코드가 이미 채워진 화상 수업 화면으로 이동합니다.
같은 컴퓨터에서 테스트하려면 다른 브라우저 탭(또는 시크릿 창)을 하나 더 열어 같은 수업 코드로 입장해보세요.

**선생님 모드**: 헤더의 "선생님 모드 (주소 수정)" 버튼 → 비밀번호(기본값 `magic777`) 입력 → 각 클래스의 화상 수업 링크, 강의 목록, 자료실 주소를 직접 수정할 수 있습니다. 이 비밀번호는 화면 코드에 그대로 노출되어 있어(개발자 도구로 보면 누구나 볼 수 있음) 진짜 보안은 아니고, 학생이 실수로 설정을 건드리지 않게 막는 정도라고 생각해주세요. 더 안전하게 하려면 나중에 서버 쪽 인증으로 옮길 수 있습니다.

## 2. 학생과 실제로 쓰려면 (배포 필요)

`localhost`는 본인 컴퓨터에서만 열립니다. 학생이 접속하려면 인터넷에 공개된 주소가 필요합니다. 무료로 시작할 수 있는 곳:

- **Render.com** (Web Service, Free/Starter) — GitHub 저장소 연결 후 `npm install` / `npm start`만 지정하면 자동 배포. HTTPS 자동 적용.
- **Railway.app** — 위와 비슷한 방식, 무료 크레딧 제공.
- **Fly.io** — `fly launch` 로 Node 앱 자동 감지 배포.

배포 후 `https://내앱.onrender.com` 같은 주소를 학생에게 공유하면 됩니다. WebRTC(카메라 접근)는 `localhost`가 아니면 **HTTPS가 필수**인데, 위 서비스들은 기본으로 HTTPS를 제공합니다.

## 3. 연결이 잘 안 될 때 (TURN 서버)

집/사무실 네트워크끼리는 대부분 별도 설정 없이(STUN만으로) 연결됩니다. 그런데 학교·회사처럼 방화벽이 엄격한 네트워크에 있는 학생은 "연결 중..."에서 멈출 수 있습니다. 이럴 때는 TURN 서버가 필요합니다.

`public/client.js` 상단의 `ICE_SERVERS` 배열에 TURN 서버 정보를 추가하세요:

```js
const ICE_SERVERS = [
  { urls: 'stun:stun.l.google.com:19302' },
  { urls: 'turn:YOUR_TURN_HOST:3478', username: 'YOUR_USER', credential: 'YOUR_PASS' },
];
```

무료/저비용으로 TURN을 구할 수 있는 곳: Cloudflare Calls TURN, Twilio Network Traversal Service, Metered.ca(무료 등급 있음).

## 4. 사용 방법

1. 상단 "랜덤" 버튼으로 수업 코드를 만들거나, 원하는 코드(예: `choi-mon`)를 직접 입력.
2. 학생에게 배포된 주소 + 같은 수업 코드를 알려줌 (또는 아래 6번의 딥링크 사용).
3. 같은 코드로 입장한 사람끼리 자동으로 화상 연결.
4. 하단 버튼: 마이크/카메라 끄고 켜기, **화면 공유**, 나가기. 우측 상단 "채팅"으로 텍스트 채팅.

화면 공유 버튼을 누르면 브라우저의 "공유할 화면/창/탭 선택" 창이 뜹니다. 선택하면 카메라 대신 그 화면이 상대방에게 보이고, 다시 누르거나 브라우저의 "공유 중지"를 누르면 카메라로 돌아옵니다.

**아이패드 등 태블릿 사용 시 참고**: 화면 공유(getDisplayMedia)는 데스크톱 Chrome/Edge/Firefox와 Mac Safari에서는 잘 동작합니다. iPadOS는 Safari 17 이상에서 지원되지만, 기기/버전에 따라 화면 공유 버튼이 아예 나타나지 않을 수 있습니다. 패드로 화면 공유가 안 되면: (1) iPadOS를 최신 버전으로 업데이트, (2) 반드시 Safari로 접속(Chrome 앱 아님), 그래도 안 되면 노트북/PC에서 화면 공유만 담당하는 방식을 권장드려요.

## 4-1. 배경 및 효과

하단 "배경" 버튼을 누르면 패널이 열리고, 없음/블러 외에 "전문적"(서재·오피스·심플)과 "아늑한 집"(카페·거실·독서공간) 두 카테고리, 총 6가지 배경 중 고를 수 있습니다. 브라우저에서 실시간으로 사람과 배경을 분리해서(구글의 MediaPipe Selfie Segmentation, 서버로 영상이 전송되지 않고 학생 컴퓨터 안에서만 처리됩니다) 고른 배경으로 바꿔치기합니다. `public/backgrounds/` 폴더의 PNG 6장(study/office/simple/cafe/living/nook)이 배경 이미지이고, 원하시면 같은 크기(1280×720)의 다른 이미지로 교체하셔도 됩니다.

참고할 점:
- **CPU를 사용합니다**: 배경 분리는 학생 컴퓨터에서 실시간으로 계산하는 작업이라 오래된 노트북/저사양 기기에서는 화면이 약간 끊길 수 있습니다. 느려지면 "없음"으로 끄면 원래 카메라로 바로 돌아갑니다.
- **화면 공유와는 동시에 안 씀**: 화면 공유 중에는 배경 효과가 잠시 꺼져 있다가, 화면 공유를 멈추면 고른 배경으로 자동으로 돌아옵니다.
- **브라우저 지원**: 데스크톱 Chrome/Edge가 가장 안정적입니다. 구형 브라우저이거나 라이브러리 로딩이 막힌 환경(사내망 등)에서는 "이 브라우저에서는 배경 기능을 사용할 수 없어요" 알림이 뜨고 일반 카메라로 계속 진행됩니다.

## 4-2. 입장 승인 (대기실)

수업 코드를 아는 사람은 누구나 바로 입장하는 대신, 선생님이 한 명씩 승인하고 싶을 때 쓰는 기능입니다.

**작동 방식**:
1. 같은 수업 코드로 **가장 먼저** 입장하는 사람이 그 방의 "선생님(호스트)"이 됩니다. 이때 입장 화면의 "입장 승인 필요 (대기실)" 체크박스를 켰는지 여부로 그 방의 승인 정책이 정해집니다.
2. 체크박스를 켜고 입장했다면, 이후 같은 코드로 들어오는 사람은 곧바로 통화에 들어오지 못하고 "입장 승인 대기 중" 화면에서 기다립니다.
3. 선생님 화면 상단에 "대기 중인 참가자" 버튼이 나타나고 숫자로 대기 인원이 표시됩니다. 버튼을 누르면 이름 목록과 승인/거절 버튼이 보입니다.
4. 승인하면 그 학생이 자동으로 통화에 연결되고, 거절하면 학생 쪽에는 "선생님이 입장 요청을 거절했어요" 메시지가 뜨고 입장 화면으로 돌아갑니다.

**참고할 점**:
- 체크박스는 **그 수업 코드의 첫 입장자에게만** 의미가 있습니다. 이미 누군가 그 코드로 방을 열어 둔 상태라면, 뒤에 들어오는 사람의 체크박스 설정과 무관하게 그 방에 이미 정해진 정책(선생님이 켠 값)을 따릅니다.
- 선생님이 수업 중 연결이 끊기면(네트워크 문제 등), 남은 참가자들은 계속 통화할 수 있고 이후 새로 들어오는 사람은 승인 없이 바로 입장하게 됩니다(대기 중이던 사람이 있었다면 자동으로 입장 처리됩니다). 승인 기능을 계속 쓰려면 방을 새로 만드는 것이 안전합니다.
- 방에 아무도 없어지면(모두 나가면) 그 수업 코드는 초기화되어, 다음에 처음 들어오는 사람이 다시 새 선생님이 되고 체크박스로 정책을 새로 정합니다.
- 대기 화면에서 "취소"를 누르면 대기열에서 빠지고 입장 화면으로 돌아갑니다.

## 5. 알아두면 좋은 점 (제한사항)

- **인원**: 방 하나에 최대 10명 (`server.js`의 `MAX_ROOM_SIZE`에서 조정 가능). 이 방식(mesh P2P)은 인원이 늘수록 각자 컴퓨터 부담이 커지므로 10명을 크게 넘기지 않는 걸 권장합니다. 더 큰 규모가 필요해지면 SFU(미디어 서버) 구조로 바꿔야 합니다.
- **비밀번호 없음**: 말씀하신 대로 방 비밀번호는 만들지 않았습니다 — 수업 코드를 아는 사람은 누구나 입장할 수 있습니다. 코드가 예측하기 쉬우면(예: `class1`) 외부에 공개하지 않는 게 좋아요. 나중에 필요해지면 추가할 수 있습니다.
- **녹화 기능 없음**: 필요하면 추가 개발 가능합니다.
- **화면 공유 음성 없음**: 화면 공유 시 그 화면의 소리(예: 영상 재생 소리)는 전달되지 않고 화면만 공유됩니다. 브라우저/OS 제약으로 안정적으로 구현하기 까다로워 MVP에서는 뺐습니다.

## 6. 기존 Google Sites(magic-english-korea)와의 관계

이 사이트가 기존 Google Sites 페이지 안에 있던 내용(제목, 5대 클래스, 자료실 허브, Q&A 폼)을 그대로 옮겨왔기 때문에, 이제 이 사이트 하나가 홈페이지 역할을 대신할 수 있습니다.

- **그대로 두기**: Google Sites는 계속 유지하고, 거기 있는 버튼/링크가 이 사이트(배포 주소)로 연결되게만 바꿔도 됩니다.
- **완전히 대체하기**: 도메인을 이 사이트로 옮기고 Google Sites는 더 이상 안 써도 됩니다. (도메인이 따로 있다면 Render 등 배포 서비스에 연결 가능)

각 클래스 카드의 "화상 수업 입장" 버튼은 이제 Google Meet가 아니라 이 사이트 안의 `class.html?room=클래스코드`로 연결됩니다(예: `class.html?room=basic`). 선생님 모드에서 이 링크를 다시 실제 Google Meet 주소로 바꾸는 것도 언제든 가능합니다.

**참고**: 원래 자료(강의 목록, 자료실)의 Google Drive 링크와 영단어 체크/모의고사/구글폼 링크는 모두 그대로 유지했습니다. 화상 수업 부분만 자체 시스템으로 바뀐 것입니다.

**Private 클래스 관련 참고**: 기존 사이트에는 Private 클래스가 학생 구분 없이 하나의 링크(`room=private`)로 되어 있습니다. Lee, Choi 학생을 위한 방을 따로 쓰고 싶으시면(동시에 각자 다른 선생님/시간에 수업하는 경우가 아니라면 지금 구조로도 문제없지만) `room=private-lee`, `room=private-choi`처럼 나눠드릴 수 있어요. 필요하시면 말씀해주세요.
